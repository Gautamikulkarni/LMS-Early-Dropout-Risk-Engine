How to use
InsightED.

This guide walks you through the platform — from signing in to running your first prediction. Each role sees a different part of the system, designed for a different kind of user.

1
Step one

Access the platform

Open the InsightED website and click Sign In or Sign Up. Enter your email and password. After logging in, the system automatically detects your role via Amazon Cognito and redirects you to the right dashboard — no manual navigation needed.

New user?
Use Sign Up to create your own account. After registration, you'll receive a verification email — enter the code to activate your account and sign in.
2
Step two

Know your role

InsightED has three distinct roles. Your role determines which dashboard you see and what you can do.

Student Personal dashboard
See your predicted dropout probability, 50-day engagement chart, concept mastery breakdown, submission behaviour, and a personalized AI-generated explanation of your learning patterns. Designed to help you understand your own study habits before they become a problem.
Teacher Class overview
Monitor all students in one view. The at-risk table highlights students above the dropout threshold. Click any student to open their full profile — behavioral trends, SHAP drivers, and an AI-generated recommendation for how to support them.
Tester ML model direct
Interact with the machine learning model directly. Enter any combination of the 14 behavioral features and get a live prediction — probability score, risk category, SHAP breakdown, and Bedrock-generated intervention. Designed for evaluators and technical reviewers.

Step three

Run your first prediction

The fastest way to see InsightED in action is through the Tester dashboard. Log in with the Tester account, then follow these steps:

01
Navigate to the Prediction Tester section in the sidebar.
02
Enter behavioral feature values — engagement levels, submission delays, accuracy scores, concept failure rates, response times. Default values are pre-filled to get you started.
03
Click Run Prediction. The request is sent to the FastAPI backend on AWS App Runner, processed by the Random Forest model, and returned within seconds.
04
View the results on the right — probability score, risk category, SHAP bar chart, and Bedrock AI recommendation.
4
Step four

Understand the results

Every prediction returns four outputs. Here is what each one means:

Dropout Probability
A percentage from 0–100% representing the model's confidence that this student is at risk of disengaging. Calibrated output from the Random Forest model.
Risk Category
Low (<40%) — progressing normally. Moderate (40–75%) — requires monitoring. High (>75%) — strong indicators of disengagement, immediate support recommended.
SHAP Feature Importance
A bar chart showing which behavioral signals contributed most to this prediction. Positive values push risk up, negative values push it down. This is why InsightED's predictions are explainable, not just a black box.
AI Intervention
A personalized recommendation generated by Amazon Nova 2 Lite via Bedrock. The prompt is built from the live SHAP values — so the advice is specific to this student's actual behavioral pattern, not templated text.
Pro tip
Try running two predictions — one with high engagement and low submission delay, and one with the opposite values. Watch how the probability score, SHAP chart, and AI recommendation all shift. This shows the model responding to real behavioral differences, not just returning a fixed output.

AI for Bharat · Built with purpose

We built InsightED because
dropout should never be a surprise.

Every student who drops out of a course leaves a trail of signals — small drops in engagement, late submissions, a streak of missed sessions. These signals exist in the data. But nobody was reading them in time. We decided to change that.

The Problem

By the time grades fall, it is already too late.

Traditional LMS platforms collect enormous amounts of behavioral data — session durations, submission timestamps, accuracy scores, tab switches, concept attempts. Yet almost none of this data is used to predict outcomes. It sits in dashboards, waiting to be manually interpreted by educators who are already stretched thin.

The result: students disengage gradually, silently, and by the time a grade reflects the problem, the window for meaningful intervention has often already closed. InsightED shifts this paradigm — from reactive monitoring to proactive, data-driven early intervention.

Why AI — and why it matters here

Our AI is load-bearing, not decorative.

A lot of AI-powered tools add a chatbot on top of existing data. InsightED is different. The ML model sits at the core of every prediction — processing 14 behavioral features engineered from raw LMS activity and producing a calibrated dropout probability score for each student.

But prediction alone is not enough. We use SHAP (SHapley Additive Explanations) to make every prediction transparent — showing exactly which behavioral signals drove the risk score. And crucially, the Bedrock prompt is constructed from live SHAP values — so the AI recommendations are grounded in the actual prediction, not generic advice.

ML Model
Random Forest with calibrated probability output. Trained on 2,000 student records with 14 engineered behavioral features. Class-balanced, cross-validated, evaluated on AUC-ROC.
Generative AI
Amazon Nova 2 Lite via Bedrock generates personalized interventions. The prompt is built from live SHAP values — so the output is grounded in the actual prediction, not generic advice.
Infrastructure

Secure, reliable, and cost-efficient by design.

Every architectural decision was made with three constraints in mind: security, reliability, and cost efficiency. We chose managed AWS services that scale to zero when idle — making InsightED viable at production scale without burning budget.

Amazon S3 Storage
Stores all student datasets, model artifacts (RF model, scaler, SHAP explainer), and the frontend as a static site. Pay-per-GB, no idle cost.
Amazon SageMaker ML Development
Used for model training, feature engineering, and experimentation. Notebook instances shut down after use — zero idle compute cost.
AWS App Runner Backend Hosting
Runs the FastAPI prediction API. Auto-scales to zero between requests. No EC2 management, no always-on servers. Only charged per request-second.
Amazon Bedrock Generative AI
Nova 2 Lite via the Converse API. Called only when a prediction is made — pure pay-per-token. No model hosting cost whatsoever.
Amazon Cognito Authentication
Handles sign-up, email verification, JWT issuance, and role-based group assignment. Free tier covers all usage at our scale.
Amazon SNS Alerts
Configured to trigger automated alerts when high-risk students are detected, enabling educators to respond immediately without manual monitoring.
Reliability & Error Handling

We planned for things going wrong.

SHAP format compatibility
The SHAP library changed its output format between versions. Our backend handles both 2D and 3D array formats — predictions never fail due to library version differences.
Model download guard
S3 model files are downloaded only if not already cached at /tmp. Prevents redundant network calls on warm App Runner instances, reducing latency and cost.
Cognito token verification
Every prediction request verifies the JWT against Cognito's public JWKS endpoint. Expired or invalid tokens are rejected before any ML computation runs.
Bedrock fallback
If Bedrock fails, the API still returns the full prediction, SHAP scores, and risk category. The generative layer never blocks the core ML output.
Null-safe frontend
Missing API fields handled gracefully — dropout_probability defaults to 0, risk_category to Unknown, shap_scores to empty object. No crashes on partial responses.
Cross-origin handling
CORS explicitly configured to allow requests from the S3-hosted frontend. No browser-level blocking regardless of HTTP or HTTPS access.
